# NID Database

YAML files in this directory define the imports/exports of PS3 system
libraries. One file per library. The nidgen CLI consumes them to:

- generate PPU assembly stub files (`.section .lib.stub.<library>`),
- build stub archives (`lib<library>_stub.a`) for link-time resolution,
- verify computed FNIDs against stored NID values,
- drive the coverage-report tool.

## Two kinds of DB files here

- **Curated DBs** at the top level (e.g. `sys_lv2.yaml`) ŌĆö hand-maintained.
  Carry full signatures, notes on deprecation, ordinals for libraries that
  import by ordinal, and cross-checked NIDs. Not every library has one.
- **Extracted DBs** under `extracted/` ŌĆö machine-generated by
  `nidgen extract <lib>_stub.a` from the reference SDK. One YAML per stub
  archive (97 of them for a 3.55-4.75-era cell SDK). NIDs are
  ground-truth; `signature` is always empty and `notes` records the
  source object name.  Regenerate with `tools/nidgen/scripts/extract-all.sh`
  (or the loop in commit message) when the reference SDK version
  changes. Curation of an extracted DB happens by lifting its exports
  into (or alongside) a top-level curated YAML and filling in signatures.

## File naming

`<library>.yaml`, matching the reference SDK's library prefix:

- `sys_lv2.yaml` ŌĆö lv2 kernel syscalls (`sys_ppu_thread_*`, `sys_spu_*`, etc.)
- `cellGcmSys.yaml` ŌĆö RSX GCM runtime
- `cellNetCtl.yaml` ŌĆö network control
- `cellPad.yaml` ŌĆö controller input
- `cellAudio.yaml` ŌĆö audio output
- `cellSysutil.yaml` ŌĆö system utility (callbacks, save data)
- `cellFs.yaml` ŌĆö filesystem
- `cellSpurs.yaml` ŌĆö SPU thread scheduler

## Schema

See `schema.yaml` for the full JSON Schema. Minimum structure:

```yaml
library: cellNetCtl            # PSL1GHT lib prefix
version: 1                     # schema version
module: sys_net                # vendor SPRX module the library lives in
exports:
  - name: cellNetCtlInit
    nid:  0xbd5a59fc           # authoritative NID from reference stub .a
    signature: "int cellNetCtlInit(void)"
  - name: cellNetCtlGetInfo
    nid:  0x8ce13854
    signature: "int cellNetCtlGetInfo(int code, CellNetCtlInfo *info)"
```

## Where do NIDs come from?

The reference SDK ships stub `.a` archives under `target/ppu/lib/`. Each
archive holds one ELF object per exported symbol plus a `_head.o` anchor.
The per-symbol objects carry a 20-byte `.psp_libgen_markfunc` section
whose bytes 8..12 store the authoritative NID as a big-endian u32. The
exported name is the lone `GLOBAL NOTYPE` symbol whose `st_shndx` points
at that section.

`.rodata.sceFNID` (4 bytes) is a placeholder ŌĆö it is zeroed in the stub
archive and patched only at SPRX-build time. Do not read NIDs from it.

`nidgen extract --input lib<name>_stub.a --output <name>.yaml` is the
canonical way to produce a fresh DB.

The `nidgen verify` command cross-checks that, for each entry, the
computed FNID (SHA-1(name || suffix)[0..4], little-endian) matches the
stored NID. The reference SDK's build pipeline originally assigns NIDs
with this same algorithm, so a mismatch means the YAML is stale or the
name was transcribed wrong ŌĆö empirically all 4981 symbols extracted
from a 3.55-4.75-era cell SDK verify cleanly.

## Legal / clean-room note

The YAML files here contain only **names, NIDs, and signatures** ŌĆö no proprietary
source code or header contents. Names are recovered from public ABI (ELF
symbol tables and exported ordinals); NIDs are computed hashes of those names.
Signatures are reimplemented from the reference header text by clean-room
paraphrase, not copy.

The reference SDK mount at `reference/private/` is never committed.

## Populating a new library

1. Extract the ground-truth DB:
   `nidgen extract --input $REF_SDK/target/ppu/lib/lib<name>_stub.a --output extracted/lib<name>_stub.yaml`
2. Verify it round-trips:
   `nidgen verify extracted/lib<name>_stub.yaml`
3. If you need signatures, lift the exports into a top-level curated YAML
   (`<name>.yaml`) and paraphrase C signatures from the reference header.
   Do not copy verbatim.
4. Emit stubs: `nidgen stub <name>.yaml > <name>.s` and
   `nidgen archive ...` to build the linkable `.a`.
